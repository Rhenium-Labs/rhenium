# Security

Rhenium takes security seriously. If you believe you’ve found a security vulnerability in Rhenium or any repository under this project, we’d appreciate you reporting it responsibly so we can investigate and fix it as quickly as possible.

## Reporting a Vulnerability

**Please do not disclose security issues** via GitHub issues, discussions, or pull requests. Instead, report vulnerabilities privately by emailing:

**📧 [contact@rhenium.app](mailto:contact@rhenium.app)**

When reporting an issue, please include as much detail as possible to help us understand and reproduce the problem:

- The type of vulnerability (for example: privilege escalation, injection, or unintended access).
- The affected file(s) or component(s).
- The location of the vulnerable code (branch, commit, tag, or direct link).
- Any configuration or setup required to reproduce the issue.
- Clear steps to reproduce the vulnerability.
- Proof-of-concept code or examples, if available.
- The potential impact and how the issue could be exploited.

Providing detailed information helps us triage and resolve issues faster.

## Response & Disclosure

We aim to acknowledge vulnerability reports within **72 hours**. If the issue is confirmed, we’ll work on a fix as quickly as possible and coordinate a responsible disclosure once a patch or mitigation is available.

Thanks for helping keep Rhenium and its users safe.
